Test methodology caveat:
  Original plan was real S3 or freeze sleep. Found:
  - S3 (systemctl suspend): QEMU ARM on Apple Silicon doesn't support
    resume. Cold-reboots instead of resuming. Evidence: boot ID changed
    in journalctl --list-boots.
  - freeze (echo freeze > /sys/power/state): VM hung indefinitely,
    required manual UTM "Restart VM" to recover.
  - SIGSTOP/SIGCONT: works. Pauses Go runtime, including the 5s ticker,
    so when the process resumes, time.Now() reports a large tick gap —
    exactly the code path we wanted to exercise.

  SIGSTOP tests the AGENT's time-jump handling, NOT the full OS-level
  sleep/wake (no network interface down, no DNS reset, no TUN driver
  pause). We had those dimensions covered by the macOS LAN + cellular
  runs. The Linux-specific gap — "does the agent's tick-gap detection
  work on Linux?" — is now closed.
